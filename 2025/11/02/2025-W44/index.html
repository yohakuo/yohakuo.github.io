  • 谈谈 AI 编程工具的进化与 Vibe Coding
    具体地讲述了 AI 辅助工具的发展历程，每个产品在各阶段有了什么突破：GitHub Copilot 的代码上下文共享和补全操作；完整 AI IDE 工具 Cursor，专有的 Tab 自动补全能力、Claude 3.5 Sonnet 以及上下文工程的一个关键突破-使用 RAG 将项目整个 codebase 进行索引，并以语义 (向量) 搜索的方式给 LLM 提供整个项目的上下文。Cursor 会在创建新项目的时候，将整个代码库在本地拆分为多个小块内容后，上传到 Cursor 云服务器，使用 embedding 模型来 embeds 代码，并存储到云向量数据库中。在 Chat/Agent 中提问时，Cursor 会在推理时对 prompt 进行嵌入，让 Turbopuffer 进行最近邻搜索，将混淆后的文件路径和行范围发送回 Cursor 客户端，并在客户端本地读取这些文件代码块。Turbopuffer 是基于对象存储从头构建的无服务器向量和全文搜索。（可以先简单理解为语义搜索）终端命令行辅助编码工具 Claude Code ，选择提供足量的上下文，并基于 Unix 工具进行检索。例如用 grep, find, git, cat 等等终端命令而不是 RAG 的方案。在用户提问之后，基于提问进行关键词不断检索，直到找到项目中所有需要的上下文代码，然后再开始编程，亦或者是一轮一轮的对话、编程和检索，一直重复这个过程，直到 LLM 认为找全了上下文。

  • 【B站视频】向量数据库技术鉴赏
    向量数据库科普。当事物被向量化，且拥有足够多的特征维度后，我们就能区分开不同的事物。向量化数据在高维的坐标空间中理应对应一个坐标，这时通过计算它们之间的距离，就能简单判断它们的相似度。后面还提到了两种优化搜索的方法：通过聚类或哈希做分类。两种方法都是在分类的基础上在更小的范围里面再搜索。这个 up 还用向量数据库做了一个开源的项目-关于正史的问答模型。
